css真的懂了吗——看似简单的问题

发表于 2017-01-20

对css的基础的掌握真的够老司机了吗？来回答下面的这些简单css问题吧

定位篇

position的取值有几种，他们之间的区别是什么？

不经过大脑回答的会说3种：relative、absolute和fixed。因为你会忘记还有一个不起眼的默认属性static。（目前兼容性极差的center、page和sticky不考虑）
四者的特点：

  • static: 遵循文档流。四个定位偏移属性不会起作用，并且不能配合z-index设置层叠级别（称为非定位元素）。
  • relative: 遵循文档流。top、bottom、left、right设置偏移时参照自身在文档流中的位置偏移，并且不会影响到文档流，可配合z-index设置层叠级别。
  • absolute: 脱离文档流。偏移属性参照离自己最近的定位祖先元素（position: relative|absolute|fixed）的左上角定位，如果没有就参照body，由于脱离文档流，margin不会与其他元素融合，可配合z-index设置层叠级别。
  • fixed: 脱离文档流。偏移属性参照设备窗口，不会跟着页面滚动时滚动，可配合z-index设置层叠级别。

那么问题来了，上面的代码中，a的哪一个部分的左上角在box1的哪一个部分的左上角对齐了呢？

2.a的content的左上角对齐了box1的padding的左上角
3.a的content的左上角对齐了box1的border的左上角

5.a的padding的左上角对齐了box1的content的左上角

如果是一道面试题，你是蒙不对的，哈哈哈。
答案是：a的margin的左上角对齐了box1的padding的左上角！。子元素的偏移起点是自身margin的四个顶点的某一个顶点，而定位父元素的坐标原点是padding的四个顶点的某一个顶点。(Chrome,Safari和Firefox亲测);

什么是层叠上下文，z-index要怎么使用？

如果说每个元素是桌子上的纸张，top、bottom、left和right是对元素x轴和y轴上的移动，那z-index就是z轴上的移动（叠放关系）。全局层叠上下文就是整张桌子（body标签），局部层叠上下文就是把它们放在同一张纸上进行叠放形成一摞纸。
概念：

  • 自定义层叠关系：z-index为正 > z-index:auto|z-index:0。 默认层叠关系：子元素 > 父元素 后面的元素 > 前面的元素

  • 定位元素（position: relative|absolute|fixed）配合z-index属性会改变自身的层叠等级并以自身为容器创建局部层叠上下文。

在没有css的情况下，以默认层叠关系：子元素 > 父元素 后面的元素 > 前面的元素。test2这个div以及它的子元素都会盖在test这个div以及它的子元素上面。
